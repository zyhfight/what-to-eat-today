# 外卖推荐对接第三方聚合平台方案

> 个人开发者无法申请美团联盟企业资质时的替代方案分析

---

## 一、问题背景

美团联盟要求**企业资质**（营业执照 + 安全合规认证），个人开发者无法直接申请 AppKey。已实现的外卖推荐功能依赖联盟 `query_coupon` 接口，需要寻找替代方案。

---

## 二、可选方案总览

| 方案 | 门槛 | 外卖商品搜索 | 佣金/收益 | 推荐度 |
|------|------|-------------|----------|--------|
| **饿了么本地生活联盟** | 个人可注册 | ✅ 有 | 6% 佣金 | ⭐⭐⭐⭐⭐ |
| **好单库聚合平台** | 个人免费注册 | ⚠️ 主要是优惠券/活动链接 | 佣金分成 | ⭐⭐⭐⭐ |
| **微客云霸王餐CPS** | 个人可接入 | ⚠️ 偏活动/霸王餐 | CPS返利 | ⭐⭐⭐ |
| **大淘客联盟** | 个人可注册 | ❌ 主做电商，外卖弱 | 佣金分成 | ⭐⭐ |

---

## 三、推荐方案详解

### 方案一：饿了么本地生活联盟（最推荐）

饿了么是阿里旗下，其 CPS 联盟对个人开发者友好：

- **注册地址**：饿了么联盟平台（union.ele.me 系列域名）
- **注册方式**：直接用饿了么账号登录，无需企业资质
- **佣金比例**：**6%**（高于美团的 3-5%）
- **接入方式**：
  - 基础：获取推广链接/海报（零开发）
  - 进阶：API 接口调用（需要申请开发者权限）
  - 可嵌入微信公众号菜单、小程序、H5 页面

**与现有架构的集成点**：

```
当前「今天吃什么」专家
  ├── 到店推荐 → click.meituan.com/cps/ai/*（已有，pt-passport 登录）
  ├── 外卖推荐 → 美团联盟 query_coupon（需企业资质，阻塞）
  └── 外卖推荐 → 饿了么联盟 API（新增，个人可接入）
```

具体改造：
1. 新增 `run.js search-eleme` 命令，调饿了么联盟商品搜索接口
2. `recommend.js` 新增 `--mode waimai --platform eleme` 分支
3. Agent 提示词增加「美团外卖 / 饿了么外卖」选择

### 方案二：好单库聚合平台

好单库是最大的第三方 CPS 聚合平台之一，**个人免费注册**：

- **注册地址**：https://www.haodanku.com/openapi/api_apply
- **支持的本地生活**：美团活动链接、饿了么活动链接（主要是优惠券/红包分发，非商品级搜索）
- **接入方式**：RESTful API，AppKey + MD5 签名
- **费用**：免费，但佣金会被平台抽成 10-30%

**局限性**：好单库对美团/饿了么的支持偏向「活动链接 + 优惠券分发」，不像官方 API 那样支持按关键词搜索附近外卖商品。适合做**领券+返利**场景，不太适合**个性化推荐**场景。

### 方案三：微客云霸王餐 CPS

- **定位**：外卖霸王餐 + 优惠券 CPS 返利
- **支持平台**：美团、饿了么
- **接入方式**：提供 API 接口
- **局限**：偏向「霸王餐活动」（用户下单后返现），而非通用商品搜索推荐

## 四、推荐实施路径

### 第一阶段：接入饿了么（1-2 周）

```
「今天吃什么」专家
  ├── 到店 → 美团（已有）
  ├── 外卖 → 饿了么（新增）
  └── 外卖 → 美团（待企业资质）
```

具体步骤：
1. 注册饿了么本地生活联盟账号
2. 研究饿了么联盟 API（商品搜索、转链、订单查询）
3. 在 `run.js` 新增 `search-eleme` 命令
4. 在 `recommend.js` 新增饿了么外卖评分算法
5. Agent 提示词新增「美团/饿了么」平台选择

### 第二阶段：补充好单库兜底（1 周）

用好好单库的美团/饿了么活动链接接口作为兜底：
- 当美团联盟 API 不可用时，降级到好单库的活动链接
- 当饿了么联盟 API 异常时，同样降级

### 第三阶段：申请美团企业资质（长期）

如果业务跑通后有流水，可以考虑注册个体户/公司申请美团联盟：
- 个体工商户即可（成本低）
- 美团联盟 AppKey 获取后，现有代码**零改动直接启用**

## 五、架构改造要点

### 新增平台适配层

```
recommend.js
  ├── --mode dinein --platform meituan  → 到店（已有）
  ├── --mode waimai --platform meituan  → 美团外卖（已有，待 AppKey）
  ├── --mode waimai --platform eleme    → 饿了么外卖（新增）
  └── --mode waimai --platform haodanku → 好单库聚合（新增，兜底）
```

### 统一数据模型

各平台返回的外卖数据拍平为统一格式：

```js
{
  platform: 'eleme' | 'meituan' | 'haodanku',
  productId, poiId, poiName, brandName,
  productName, sellPrice, imageUrl,
  deliveryDistance, distributionCost,  // 配送费
  deliveryDuration,                    // 配送时长
  saleVolume,                          // 销量
  h5Url, deeplink                      // 推广链接
}
```

评分算法保持现有外卖公式不变，各平台共用。

---

## 六、一句话总结

**饿了么本地生活联盟是个人开发者做外卖推荐的最佳切入点**——个人可注册、有正式 API、佣金 6%、接入成本低。好单库作为补充兜底。美团外卖推荐代码已就绪，拿到企业 AppKey 后零改动直接启用。
