# 「今天吃什么」专家 — 功能与技术方案总结

> 一个 AI 驱动的美食推荐 Agent，支持堂食推荐 + 外卖推荐 + 红包分发，零门槛可用

---

## 一、功能总览

### 1.1 堂食推荐（到店用餐）

| 能力 | 说明 |
|------|------|
| 触发方式 | 「今天吃什么」「帮我推荐附近好吃的」等 |
| 偏好收集 | 时段自动判断 + 口味/菜系/预算/忌口收集 |
| 搜索方式 | 美团 CPS 接口，关键词 + 经纬度实时搜索附近商家 |
| 评分排序 | 评分(40%) + 距离(30%) + 预算匹配(20%) + 偏好(10%) |
| Top3 展示 | 商品名、价格、距离、评分、图片、推荐理由 |
| 一键下单 | 美团团购券直接下单，返回支付二维码 |
| 登录方式 | 美团 pt-passport 扫码登录，token 持久化 |

### 1.2 外卖推荐

| 能力 | 说明 |
|------|------|
| 触发方式 | 「外卖」「不想出门」「配送」等关键词自动识别 |
| 搜索方式 | 三层降级：美团联盟 → 饿了么商品池 → **到店搜索外卖品类**（兜底） |
| 评分排序 | 外卖版：配送费/时长/销量，或品类版：评分/距离/价格 |
| Top3 展示 | 商品信息 + 美团 H5 下单链接 + 红包入口 |
| 红包分发 | 喵有券聚合平台，美团/饿了么红包转链 |

### 1.3 红包品类推荐（兜底中的兜底）

| 能力 | 说明 |
|------|------|
| 触发方式 | 外卖搜索全部不可用时的最终降级 |
| 推荐方式 | 品类引导（火锅/快餐/奶茶…）+ 美团红包链接 |
| 用户流程 | 领红包 → 自行去美团 App 选品下单 |

## 三、降级链（核心设计）

外卖推荐按以下优先级自动降级，确保**任何情况下都有结果**：

```
用户说「外卖推荐」
  ├─ 1️⃣ 美团联盟 query_coupon（需企业 AppKey）
  │     └─ ❌ → 降级
  ├─ 2️⃣ 饿了么联盟 itempromotion.query（需淘宝客 AppKey）
  │     └─ ❌ → 降级
  ├─ 3️⃣ 美团 CPS searchProductByAgent 搜外卖品类（✅ 已有，零门槛）
  │     └─ ✅ 搜到火锅/麻辣烫/炸鸡/奶茶/披萨等
  └─ 4️⃣ 喵有券红包推荐（✅ 已调通）
        └─ ✅ 品类引导 + 美团红包链接
```

## 四、评分算法

### 到店评分
```
score = 0.40 × (评分/5)
      + 0.30 × (1 - 距离km/8)
      + 0.20 × 预算匹配度
      + 0.10
```

### 美团外卖评分（需 AppKey）
```
score = 0.25 × (评分/5)
      + 0.25 × (1 - 配送时长/60)
      + 0.20 × 预算匹配度
      + 0.20 × (1 - 配送费/15)
      + 0.10 × log10(销量+1)/4
```

### 饿了么评分
```
score = 0.35 × 佣金性价比
      + 0.25 × 销量归一化
      + 0.25 × 预算匹配度
      + 0.15 × 折扣力度
```
